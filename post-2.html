Network Segmentation Strategy

This deployment follows a robust network segmentation model, isolating resources across subnets for enhanced security and manageability:

Figure 2: Private subnet 10.0.2.0/24 configuration

Key Design Decisions

  • Public Subnet (10.0.1.0/24): Hosts the OwnCloud web application
  • Private Subnet (10.0.2.0/24): Contains the backend MySQL database
  • NAT Gateway: Ensures secure internet-bound access for private subnet resources

Security Configuration

Network Security Groups

NSGs enforce the principle of least privilege, strictly controlling inbound and outbound traffic:

AppNSG Rules

  • SSH (22) - Limited to jump host access only
  • HTTP (80) - Standard web traffic
  • HTTPS (443) - Encrypted traffic over TLS

DbNSG Rules

  • MySQL (3306) - Accessible only from AppNSG
  • SSH (22) - Restricted to bastion host

Secure Outbound Connectivity

Outbound internet access for private VMs is routed via a NAT Gateway, which enhances both security and control:

  • No Public IPs: Internal VMs operate without direct exposure to the internet
  • Stateful Firewall: Enables return traffic for outbound sessions only
  • Port Preservation: Maintains original source ports for compatibility and traceability

Secure SSH Access

Access to backend servers is tightly secured through a bastion host and key-based SSH authentication:

  • Key-Based Authentication: Disables password-based login to reduce brute-force risk
  • Bastion Host Pattern: All SSH sessions are routed through a hardened jump server
  • Permission Lockdown: File permissions for SSH keys are strictly enforced
